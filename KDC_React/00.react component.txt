[React 컴포넌트의 종류]
1.클래스형
2.함수형

*2019년 v16.8부터 함수형 컴포넌트에 훅(Hook) 지원 시작
*현재는 공식문서에서도 함수형 컴포넌트와 훅(Hook)을 함께 사용하는 것을 권장 

[클래스형 vs 함수형]
*클래스형	컴포넌트 내에서 로직과 상태를 구현
*함수형  		state를 props로 받아와 UI에 뿌려준다
		로직의 재사용 가능(장점)

[컴포넌트의 목적에 따른 분류]
1.프레젠테이셔널 컴포넌트
	- view 담당(ui)
	- 프레젠테이셔널 컴포넌트 내부에서는 프레젠테이셔널 컴포넌트와 컨테이너 컴포넌트를 모두 사용할 수 있다
	- 리덕스에 직접 접근X, props로만 데이터, 함수를 가져온다
	- state를 사용하지X, 만약 있다면 데이터가 아닌 UI에 대한 state여야 한다
	- 주로 함수형 컴포넌트로 작성 
2.컨테이너 컴포넌트
	- 다른 프레젠테이셔널 컴포넌트나 컨테이너 컴포넌트를 관리한다
	- 내부에 DOM 엘레먼트를 작성하지 않는다(사용하는 경우는 감싸는 용도)
	- 스타일X, 스타일은 프레젠테이셔널 컴포넌트 내부에 정의한다
	- state를 가지고 있으며, 리덕스에 직접 접근하여 데이터를 가져온다
	- 내부에 dispatch를 하는 함수를 구현한다 (참고: https://mjn5027.tistory.com/33)
